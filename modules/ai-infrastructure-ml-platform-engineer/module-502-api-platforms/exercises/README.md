# Module 2 Exercises: API Design for ML Platforms

This directory contains 6 hands-on exercises to master API design patterns for ML platforms.

## Exercise Overview

| Exercise | Title | Difficulty | Duration | Key Skills |
|----------|-------|------------|----------|------------|
| 01 | Implement API Versioning | Intermediate | 60 min | Versioning strategies, backward compatibility |
| 02 | Build gRPC Model Serving | Advanced | 120 min | Protocol Buffers, gRPC, streaming |
| 03 | Add Rate Limiting & Throttling | Intermediate | 75 min | Rate limiting, Redis, token bucket |
| 04 | Create Production SDK | Advanced | 180 min | SDK design, error handling, retries |
| 05 | Generate OpenAPI Documentation | Basic | 45 min | OpenAPI spec, Swagger UI, client generation |
| 06 | API Performance Benchmarking | Intermediate | 90 min | Load testing, profiling, optimization |

## Prerequisites

- [x] Completed Module 1 exercises
- [x] Understanding of HTTP protocols
- [x] Experience with REST APIs
- [x] Python async/await familiarity
- [x] Basic Protocol Buffers knowledge

**Total Time**: 570 minutes (~9.5 hours)
